Tools and Practice for Better, Faster, Easier Debugging

About the Event

Software often breaks in ways that aren't obvious, and debugging under tight deadlines can be one of the most frustrating parts of coding. Whether you're struggling with runtime errors, hidden logic flaws, or just want to stop "guess-and -check" debugging, this session is for you.

Join us as we explore practical tools and proven techniques that help developers debug smarter and faster. You will lwaen how to use breakpoints, stack traces, IDE tools, print-debugging strategies, and more-whether you're using Java, Python, or web technologies. Bring your laptop, your curiosity, and your buggy code!

What to Expect

Debug smarter, not harder --tools & strategies that save time and build confidence

  • Understand Your Errors: Your Learn how to identify and interpret runtime and logic errors quickly and clearly.
  • Master Your Tools: Discover debugging features in popular IDEs like IntelliJ, VS Code, and Eclipse that can help you step through code and inspect variables in real time.
  • Develop Efficient Habits: Build a personal debugging workflow that minimizes time spent stuck and maximizes problem-solving efficiency.

Event Schedule

Student Center Building, Room-120

Friday Evening - Introduction & Debugging Mindset

Kick off the workshop with an overview of common software bugs and real-world examples. Learn why debugging is a core skill every developer needs and how to build a problem-solving mindset.

Saturday Evening - Hands-On Debugging Practice

Work in small groups to identify and fix bugs in sample Java, Python, or HTML/JS files. You'll use breakpoints, print statements, and debugging tools inside IDEs like VS Code and IntelliJ to solve real coding challenges.
